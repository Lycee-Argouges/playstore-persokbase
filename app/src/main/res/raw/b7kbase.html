Diagnostiquer un problème d'impression

Il est rare de trouver une imprimante USB connectée à un ordinateur de bureau dans le cadre de l’environnement informatique d’une entreprise. Il est encore plus rare que celle-ci soit partagées sur le réseau sur le mode pair-à-pair. Aujourd’hui, les imprimantes sont gérées depuis des consoles dans le Cloud au travers du protocole IPP (Internet Printing Protocol).

L’interface HTTP

Chaque imprimante dotée d’un port Ethernet a une interface HTTP à partir de laquelle les informations détaillées de l’imprimante sont rendus accessibles. Depuis un poste client, entrez l’adresse IP de l’imprimante dans un navigateur Web pour consulter, par exemple, le niveau de l’encre.

Le rapport d’erreur

Si l’imprimante est inaccessible depuis un navigateur Web, rendez-vous physiquement à l’imprimante pour imprimer le rapport d’état TCP/IP. Sinon, désinstallez/réinstallez l’imprimante à partir du panneau de configuration du PC. Choisissez les pilotes d’impression universels pour une utilisation standard, et les pilotes d’impression du matériel si les options avancées de l’imprimante vous intéressent.

La connexion TCP/IP

Dans le domaine pédagogique, un serveur d’impression a la charge d’installer automatiquement les imprimantes sur les postes clients en distribuant les pilotes prédéfinis à chaque nouvelle demande d’installation. Tous les routeurs sont configurés pour autoriser le protocole IPP et router les paquets TCP avec une qualité de service garantissant la connexion continue des imprimantes au serveur.

Vers le numérique

La gestion externalisée de la maintenance des imprimantes nécessite que la société spécialisée accède aux rapports d’erreur depuis le Web. L’ouverture du pare-feu de l’établissement en accès HTTP étant proscrit, le protocole IPP transfert les données de l’imprimante sur le Cloud avec une sécurité garantie. Cette prérogative facilite les négociations dans les contrats de maintenance.
