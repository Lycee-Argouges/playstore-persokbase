Diagnostiquer une panne de réseau

Un ordinateur qui n'est pas relié au réseau Ethernet aura des voyants lumineux éteints. Si la prise est bien brassée, et que l'ordinateur démarre, différents outils sont à votre disposition.

Panneau de configuration du réseau

La première chose à vérifier est l'adresse IP de l'ordinateur. Elle doit avoir la forme 172.20.X.X : cela se comprend par l'adressage effectué par le serveur DHCP de l'établissement. Le Dynamic Host Configuration Protocol distribue un identifiant (IP), un masque (sous-réseau), et une passerelle (Internet) afin que chaque ordinateur soit connu des autres. Le sous-réseau 172.16.64.X est bloqué pour la navigation sur internet.

Système et sécurité

La communication sur Internet s'effectue en plusieurs transactions TCP/IP : un message de sortie réserve un socket E/S sur la carte ; le serveur répond sur un port logiciel ouvert dans le pare-feu (firewall) du PC ; les routeurs déterminent la route la plus sûre pour acheminer les messages. Vérifiez que la connexion utilisée appartient aux règles du pare-feu "Domaine/Bureau" ou "Privé/Domestique". N'utilisez le mode "Public" que lorsque vous ne connaissez pas votre environnement réseau.

Invite de commande

Les commandes ci-dessous sont indispensables :

Ipconfig /all

 Informe l'utilisateur des propriétés de la carte réseau. Pour chaque carte réseau détecté, même virtuelle, vous récupérez votre adresse MAC, votre adresse IP, votre masque de sous-réseau et votre passerelle.

Ipconfig /release

 Vous libérer l’adresse IP retenue par la carte réseau. La commande ipconfig /renew s’utilise en complément, pour demander au serveur DHCP une nouvelle adresse IP.

Ipconfig /displaydns

 Affiche le contenu du cache de résolution DNS. Chaque site dispose d’une adresse IP et d’un nom convivial, dont la résolution (translation) est gérée par le Dynamic Name Service.

Ping destination –n X

 Détermine l'état de la connexion vers un site distant. Elle se base sur le protocole ICMP qui ne transmet pas de données sur le PC demandant l'information.

Arp -a

 Affiche la correspondance entre adresses IP et adresse MAC (physique) des ordinateurs connectés au même sous-réseau que le PC utilisé, et dont la connexion pair-à-pair a été initialisée.

Tracert

 Permet de suivre la progression d'un message sur le réseau à destination d'un site, affichant les adresses IP des matériels par lesquels ce message transite.

Route

 Route print affiche les itinéraires persistants sur un ordinateur. C'est-à-dire si l'ordinateur prédéfini le trajet d'un message qu'il envoie à destination d'un site.

Netstat

 Informe l'utilisateur sur les ports logiciels ouverts/fermés/en attente du PC utilisé.

Paramètres internet

Enfin, réinitialisez les paramètres internet en cas de connexion impossible depuis un poste de travail particulier. Certains sites demandent la modification des paramètres avancés d'Internet Explorer et causent des erreurs d'exécution du programme. De manière générale, testez une connexion avec un navigateur web dont les paramètres ont été réinitialisé avant d'étudier un dysfonctionnement du réseau.
